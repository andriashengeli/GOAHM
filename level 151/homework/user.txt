API  - ძირითადად გამოიყენება იმისთვის რომ ჩვენ დავუკავშირდეთ სერვერს 
API ნიშნავს აპლიკაციის პროგრამირების ინტერფეისს. API-ების კონტექსტში, სიტყვა „აპლიკაცია“ აღნიშნავს ნებისმიერ პროგრამულ უზრუნველყოფას, რომელსაც აქვს განსხვავებული ფუნქცია. ინტერფეისი შეიძლება განვიხილოთ, როგორც მომსახურების კონტრაქტი ორ აპლიკაციას შორის.


სულ არსებობს 4 ტიპის api 
1. საჯარო API-ები
საჯარო API ღიაა და ხელმისაწვდომია ნებისმიერი გარე დეველოპერის ან ბიზნესის მიერ გამოსაყენებლად. საწარმო, რომელიც შეიმუშავებს ბიზნეს სტრატეგიას, რომელიც გულისხმობს მისი აპლიკაციებისა და მონაცემების სხვა ბიზნესებთან გაზიარებას, შეიმუშავებს და შესთავაზებს საჯარო API-ს. მათ ასევე უწოდებენ ღია API-ებს ან გარე API-ებს.

2. პარტნიორის API-ები
პარტნიორის API, რომელიც ხელმისაწვდომია მხოლოდ სპეციალურად შერჩეული და ავტორიზებული გარე დეველოპერებისთვის ან API მომხმარებლებისთვის, წარმოადგენს ბიზნესებს შორის აქტივობების ხელშეწყობის საშუალებას.

3. შიდა API-ები
შიდა ან კერძო API განკუთვნილია მხოლოდ საწარმოში გამოსაყენებლად, რათა დააკავშიროს სისტემები და მონაცემები ბიზნესში. მაგალითად, შიდა API-მ შეიძლება დააკავშიროს ორგანიზაციის ხელფასების და ადამიანური რესურსების სისტემები.

4. კომპოზიტური API-ები
კომპოზიტური API-ები, როგორც წესი, აერთიანებს ორ ან მეტ API-ს, რათა შექმნას დაკავშირებული ან ურთიერთდამოკიდებული ოპერაციების თანმიმდევრობა. კომპოზიტური API-ები შეიძლება სასარგებლო იყოს რთული ან მჭიდროდ დაკავშირებული API ქცევების მოსაგვარებლად და ზოგჯერ შეუძლია გააუმჯობესოს სიჩქარე და მუშაობა ინდივიდუალურ API-ებთან შედარებით.


როგორ მუშაობს api 
1 - ეტაპი არის REQUEST ანუ ჩვენ ვაგზავნით რაღაც ინფორმაცია სერვერთან 
2 - ეტაპი არის სერვერი ჩვენს მიერ გადაცემულ კითხვას ამუშავებს 
3 - ეტაპი არის უკვე სერვერის მიერ დამუშავებულ ინფორმაციას გადმოაგზავნის
4 - ეტაპი არის უკვე ამ გადმოგზავნილ ინფორმაციას მომხმარებელი მიიღებს
